# üîç How to Fix the "Error loading data" Issue

## What's Happening?
The dashboard shows "Error loading data. Please refresh." because one of the API calls is failing.

## How to Diagnose

### Step 1: Check Session
Open: **http://localhost/To-Do List/session-debug.html**

This will show you:
- ‚úì If you're logged in
- ‚úì If the server recognizes your session
- ‚úì Which API endpoints are failing

### Step 2: Read the Results

When you open session-debug.html, look for:

**Green ‚úì = Good**
- "Session Check: LOGGED IN"
- "getUserStats: SUCCESS"
- "getHabits: SUCCESS"
- "getTasks: SUCCESS"
- "Dashboard would load successfully!"

**Red ‚úó = Problem**
- "Session Check: NOT LOGGED IN" ‚Üí Need to log in again
- "getUserStats: FAIL" ‚Üí Database issue
- "getHabits: FAIL" ‚Üí Database issue
- "getTasks: FAIL" ‚Üí Database issue

### Step 3: What to Do Based on Results

**Problem: "NOT LOGGED IN"**
1. Go to: http://localhost/To-Do List/auth.php
2. Log out completely
3. Log back in
4. Try dashboard again

**Problem: API says "FAIL"**
1. Check if MySQL is running
2. Check if Apache is running
3. Go to: http://localhost/To-Do List/quick-check.php
4. Look for any red ‚úó marks
5. Fix what's shown as failing

**Everything shows SUCCESS but dashboard still fails**
1. Open browser console (F12 ‚Üí Console tab)
2. Look for error messages
3. Refresh the page
4. Take a screenshot of errors if it persists

## Most Common Fixes

1. **Session expired**: Log out and log back in
2. **MySQL not running**: Start MySQL in XAMPP
3. **Apache not running**: Start Apache in XAMPP
4. **Session cookies disabled**: Check browser privacy settings
5. **Database tables missing**: Run CREATE_TABLES.sql
